### 📘 Chapter: Two Pointers  
### 📌 Problem 10: 4Sum (LeetCode 18)

---

**Input**  
- `nums`: an integer array of length `n`.  
- `target`: an integer.

**Output**  
- A list of **all unique quadruplets** `[nums[a], nums[b], nums[c], nums[d]]` such that:  
  - `0 <= a, b, c, d < n`, all distinct indices.  
  - `nums[a] + nums[b] + nums[c] + nums[d] == target`.  
- No duplicate quadruplets in the output.

**Constraints**  
- `1 <= nums.length <= 200`  
- `-10⁹ <= nums[i] <= 10⁹`  
- `-10⁹ <= target <= 10⁹`

**Example**  
```
Input:  nums = [1, 0, -1, 0, -2, 2], target = 0
Output: [[-2, -1, 1, 2], [-2, 0, 0, 2], [-1, 0, 0, 1]]

Input:  nums = [2, 2, 2, 2, 2], target = 8
Output: [[2, 2, 2, 2]]

Input:  nums = [1, 2, 3], target = 6
Output: []  (need at least 4 elements)
```

**Follow-up**  
- Generalize to `kSum`. The optimal for 4Sum is O(n³) with two pointers. An O(n²) approach exists with HashMap but is complex.

---

### 🧠 Core Idea

4Sum is a direct extension of **3Sum** (which extends **Two Sum II**).

- **Brute force:** Four nested loops check all O(n⁴) quadruplets. Too slow for n=200.
- **Sorting + Two Pointers (optimal for interview):** Sort the array. Fix the first two elements with nested loops (O(n²)), then use two pointers for the remaining two (O(n)). Total: **O(n³)**. Duplicate handling via adjacent skipping.
- **HashMap of pair sums:** Store all O(n²) pair sums, then find complementary pairs. O(n²) average time but O(n²) space and complex deduplication.

**Key insight:** Sorting enables both the two-pointer technique AND trivial duplicate skipping (identical values become adjacent).

**Overflow warning:** With values up to ±10⁹, the sum of four values can reach ±4×10⁹, exceeding `int` range. Use `long` for sum comparisons.

# 🔨 SECTION 1: BRUTE FORCE APPROACH

---

## 1A. Four Nested Loops + HashSet for Dedup (O(n⁴) Time)

**Idea:** Try every quadruplet `(i, j, k, l)` with `i < j < k < l`. If the sum equals target, sort the quadruplet and add to a `Set<List<Integer>>` for deduplication.

**Time:** O(n⁴) — C(n,4) quadruplets, each sorted in O(1) (fixed size 4).  
**Space:** O(k) — HashSet stores unique quadruplets.

```java
import java.util.*;

public List<List<Integer>> fourSum(int[] nums, int target) {
    Set<List<Integer>> set = new HashSet<>();
    int n = nums.length;

    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            for (int k = j + 1; k < n; k++) {
                for (int l = k + 1; l < n; l++) {
                    long sum = (long) nums[i] + nums[j] + nums[k] + nums[l];
                    if (sum == target) {
                        List<Integer> quad = Arrays.asList(nums[i], nums[j], nums[k], nums[l]);
                        Collections.sort(quad);
                        set.add(quad);
                    }
                }
            }
        }
    }

    return new ArrayList<>(set);
}
```

## 2A. Sorting + Nested Loops + Two Pointers (O(n³) Time, O(1) Space) ✅

**Idea:**  
1. **Sort** the array.  
2. **Outer loop `i`** (first element of quadruplet): `0` to `n-4`. Skip duplicates.  
3. **Inner loop `j`** (second element): `i+1` to `n-3`. Skip duplicates.  
4. **Two pointers** `left = j+1`, `right = n-1`: find pairs summing to `target - nums[i] - nums[j]`.  
   - If sum matches: add quadruplet, skip duplicates for `left` and `right`, move both inward.  
   - If sum < needed: `left++`.  
   - If sum > needed: `right--`.  
5. Use `long` for all sum computations to avoid overflow.

**Time:** O(n³) — outer loops O(n²), inner two-pointer O(n).  
**Space:** O(1) extra (ignoring output; sorting uses O(log n) stack).

```java
import java.util.*;

public List<List<Integer>> fourSum(int[] nums, int target) {
    List<List<Integer>> result = new ArrayList<>();
    if (nums == null || nums.length < 4) return result;

    Arrays.sort(nums);
    int n = nums.length;

    for (int i = 0; i < n - 3; i++) {
        // Skip duplicate first element
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        // Early termination: if smallest possible sum > target, break
        if ((long) nums[i] + nums[i+1] + nums[i+2] + nums[i+3] > target) break;
        // Early skip: if largest possible sum < target, continue
        if ((long) nums[i] + nums[n-1] + nums[n-2] + nums[n-3] < target) continue;

        for (int j = i + 1; j < n - 2; j++) {
            // Skip duplicate second element
            if (j > i + 1 && nums[j] == nums[j - 1]) continue;

            // Early termination for inner loop
            if ((long) nums[i] + nums[j] + nums[j+1] + nums[j+2] > target) break;
            if ((long) nums[i] + nums[j] + nums[n-1] + nums[n-2] < target) continue;

            int left = j + 1, right = n - 1;
            long target2 = (long) target - nums[i] - nums[j];

            while (left < right) {
                int sum = nums[left] + nums[right];

                if (sum == target2) {
                    result.add(Arrays.asList(nums[i], nums[j], nums[left], nums[right]));
                    // Skip duplicates for left
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    // Skip duplicates for right
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    left++;
                    right--;
                } else if (sum < target2) {
                    left++;
                } else {
                    right--;
                }
            }
        }
    }

    return result;
}
```

#### Outer: i = 0, nums[i] = -2

*(i=0, no duplicate check needed)*  
*Early check: -2 + (-1) + 0 + 0 = -3 ≤ 0 ✅ (don't break)*

##### Inner: j = 1, nums[j] = -1

*(j=1, j > i+1? 1 > 1? No → no duplicate check)*  
*target2 = 0 - (-2) - (-1) = 3*

|------|-------|--------------------------|--------------|--------|
| 2 | 5 | 0 + 2 = 2 | 2 < 3 | left++ |
| 3 | 5 | 0 + 2 = 2 | 2 < 3 | left++ |
| 4 | 5 | 1 + 2 = 3 | **3 == 3** ✅ | **ADD [-2, -1, 1, 2]**. Skip dups: none. left=5, right=4 → STOP. |

##### Inner: j = 2, nums[j] = 0

*(i=1, nums[1]==nums[0]? -1 == -2? No → no skip)*

*Early check: -1 + 2 + 1 + 0 = 2 ≥ 0 ✅*

*target2 = 0 - (-1) - 0 = 1*

| left | right | nums[left] + nums[right] | vs target2=1 | Action |
|------|-------|--------------------------|--------------|--------|
| 3 | 5 | 0 + 2 = 2 | 2 > 1 | right-- |

### 🔍 Duplicate Skipping — Four Levels

| Level | Where | Condition | Example |
|-------|-------|-----------|---------|
| First element (`i`) | Outer loop | `i > 0 && nums[i] == nums[i-1]` → skip | If sorted has two -2s, second -2 is skipped |
| Second element (`j`) | Inner loop | `j > i+1 && nums[j] == nums[j-1]` → skip | If two 0s at j=2,3, the j=3 is skipped |
| Third element (`left`) | After match | `while (left < right && nums[left] == nums[left+1]) left++` | Skip repeated values at left pointer |
| Fourth element (`right`) | After match | `while (left < right && nums[right] == nums[right-1]) right--` | Skip repeated values at right pointer |

---

### 🔍 Early Termination / Pruning Optimizations

```java
// For outer loop i:
// If the SMALLEST possible sum with i exceeds target → no solution from here
if ((long)nums[i] + nums[i+1] + nums[i+2] + nums[i+3] > target) break;

// If the LARGEST possible sum with i is still below target → skip this i
if ((long)nums[i] + nums[n-1] + nums[n-2] + nums[n-3] < target) continue;

// For inner loop j (same logic):
if ((long)nums[i] + nums[j] + nums[j+1] + nums[j+2] > target) break;
if ((long)nums[i] + nums[j] + nums[n-1] + nums[n-2] < target) continue;
```

> 📌 These prunings can dramatically reduce runtime on sorted arrays with large gaps, though worst-case remains O(n³).

### 🔍 Overflow Handling

```
nums[i] can be up to 10⁹.
Sum of four: up to 4 × 10⁹ = 4,000,000,000.
Integer.MAX_VALUE = 2,147,483,647.

∴ Sum of four values CAN overflow int!

Solution: Cast to long before adding:
  long target2 = (long) target - nums[i] - nums[j];
  // or
  long sum = (long) nums[i] + nums[j] + nums[left] + nums[right];
```

## 2B. HashMap of Pair Sums (O(n²) Average Time, O(n²) Space)

**Idea:**  
1. Compute all O(n²) pair sums and store them in a `HashMap<Integer, List<int[]>>` mapping sum → list of index pairs.  
2. For each pair `(i, j)`, look up `target - nums[i] - nums[j]` in the map.  
3. For each matching pair `(k, l)` where all indices are distinct, form a quadruplet.  
4. Use a `Set<List<Integer>>` (sorted) for deduplication.

**Time:** O(n²) average for building + querying the map.  
**Space:** O(n²) for storing all pairs.  
⚠️ Deduplication is complex; code is significantly longer and bug-prone.

```java
import java.util.*;

public List<List<Integer>> fourSum(int[] nums, int target) {
    int n = nums.length;
    Arrays.sort(nums);
    Set<List<Integer>> resultSet = new HashSet<>();
    // Map: sum → list of index pairs
    Map<Long, List<int[]>> pairSums = new HashMap<>();

    // Build all pair sums
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            long sum = (long) nums[i] + nums[j];
            pairSums.computeIfAbsent(sum, k -> new ArrayList<>()).add(new int[]{i, j});
        }
    }

    // Find complementary pairs
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            long complement = (long) target - nums[i] - nums[j];
            List<int[]> pairs = pairSums.get(complement);
            if (pairs != null) {
                for (int[] pair : pairs) {
                    // Ensure all four indices are distinct and ordered
                    if (pair[0] > j) {  // pair comes AFTER (i,j)
                        List<Integer> quad = Arrays.asList(nums[i], nums[j], nums[pair[0]], nums[pair[1]]);
                        resultSet.add(quad);  // already sorted since nums is sorted and i<j<pair[0]<pair[1]
                    }
                }
            }
        }
    }

    return new ArrayList<>(resultSet);
}
```

> ⚠️ This approach is **theoretically faster** (O(n²)) but uses O(n²) space and requires careful index management. For n=200, O(n³) = 8M ops is already fast. The HashMap approach is rarely preferred in interviews due to complexity.

# 📊 SECTION 3: TRADE-OFFS & COMPARISONS

---

## Brute Force vs Two Pointers vs HashMap

| Metric | Four Nested Loops | Sort + Two Pointers | HashMap Pair Sums |
|--------|-------------------|--------------------|--------------------|
| Time | O(n⁴) | **O(n³)** | O(n²) average |
| For n=200 | ~6.5×10⁷ | **~8×10⁶** | ~4×10⁴ |
| Space | O(k) for set | **O(1)** | O(n²) |
| Dedup method | Sort quad + HashSet | Skip adjacent duplicates | HashSet of sorted quads |
| Overflow risk | Must use `long` | Must use `long` | Must use `long` |
| Code complexity | Simple but slow | **Moderate, clean** | Complex, bug-prone |
| Interview value | Baseline only | **Gold standard** | Bonus mention |

---

## Two Pointers vs HashMap (Head-to-Head)

| Metric | Sort + Two Pointers | HashMap Pair Sums |
|--------|--------------------|--------------------|
| Time | O(n³) | O(n²) |
| Space | **O(1)** | O(n²) — up to 40,000 pairs for n=200 |
| Deduplication | **Elegant** (skip adjacent) | Complex (need Set + index checks) |
| Code length | ~30 lines | ~40+ lines |
| Bug risk | Low | High (index reuse, ordering) |
| When preferred | **Always for n ≤ 200** | Only if n is very large (10⁴+) |
| Generalizes to kSum? | ✅ (recursive k-2 outer loops) | Harder to generalize |

**Verdict:** For n ≤ 200, O(n³) = 8M operations is trivially fast. The two-pointer approach is cleaner, uses less space, and is far easier to get right under interview pressure.

## Relationship to 3Sum and 2Sum

| Problem | Approach | Time |
|---------|----------|------|
| 2Sum (sorted) | Two pointers | O(n) |
| 3Sum | Fix 1 + Two pointers | O(n²) |
| **4Sum** | **Fix 2 + Two pointers** | **O(n³)** |
| kSum | Fix k-2 + Two pointers | O(n^(k-1)) |

> 📌 Each additional "Sum" adds one more outer loop. The pattern is: **(k-2) nested loops + two-pointer inner**.

### 🎯 What to Present to the Interviewer

1. **Recognize** 4Sum as an extension of 3Sum: "We fix two elements and reduce to a two-pointer pair search."
2. **Propose sorting** to enable two pointers and trivial duplicate skipping.
3. **Walk through the algorithm:**
   - Outer loop `i` (first element). Skip if `nums[i] == nums[i-1]`.
   - Inner loop `j` (second element). Skip if `nums[j] == nums[j-1]`.
   - Two pointers `left`, `right` for the remaining sum.
   - On match: add quadruplet, skip duplicates for both pointers, move inward.
4. **Emphasize overflow:** "Since values can be ±10⁹, the sum of four can exceed int range. I'll use `long` for all sum computations."
5. **Walk through** the example `[-2,-1,0,0,1,2], target=0`:
   - i=0(-2), j=1(-1): target2=3. Two-pointer finds 1+2=3. ADD [-2,-1,1,2].
   - i=0(-2), j=2(0): target2=2. Two-pointer finds 0+2=2. ADD [-2,0,0,2].
   - i=1(-1), j=2(0): target2=1. Two-pointer finds 0+1=1. ADD [-1,0,0,1].
6. **Mention early termination:** "If the four smallest remaining values exceed target, break. If the four largest are below target, skip."
7. **State complexity:** O(n³) time, O(1) extra space.
8. **If asked to generalize:** "For kSum, use recursion: fix one element, recurse for (k-1)Sum, base case is 2Sum with two pointers."
9. **If asked about O(n²):** "A HashMap storing all pair sums can achieve O(n²) average time, but requires O(n²) space and complex index management. For n=200, O(n³) is perfectly fast."

**One‑sentence summary:**  
*Sort the array, fix the first two elements with nested loops (skipping duplicates), then use two pointers to find the remaining pair that completes the target sum — achieving O(n³) time and O(1) space with clean duplicate handling via adjacent skipping.*
