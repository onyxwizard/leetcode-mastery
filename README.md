## 🤖 How the Auto-Ticker Works (Read First!)

This repository is powered by a **GitHub Action** that automatically checks off problems and updates the progress bars whenever you push code.

**You do NOT need to manually edit this README or use special commit messages.** The automation is driven entirely by your **file names**.

### 📝 The Golden Rule of File Naming:

When you solve a problem, name your file using the **exact name of the problem** (ignoring spaces, underscores, hyphens, and case).

| Problem in README     | ✅ Valid File Names                                | ❌ Invalid File Names           |
| :-------------------- | :------------------------------------------------- | :------------------------------ |
| `Two Sum`             | `Two_Sum.py`, `two-sum.java`, `TwoSum.cpp`         | `solution1.py`, `array_prob.js` |
| `Trapping Rain Water` | `Trapping_Rain_Water.py`, `trapping-rain-water.go` | `rain.py`, `hard_prob.cpp`      |
| `LRU Cache`           | `LRU_Cache.java`, `lru-cache.py`                   | `cache.py`, `design.py`         |

**Workflow:**

1. Solve the problem and save it (e.g., `01_Arrays/Two_Sum.py`).
2. Commit and push with **any message you want**: `git commit -m "Solved Two Sum"`
3. The GitHub Action triggers, reads your file name, finds the matching checkbox below, ticks it, and updates the progress table automatically!

## 📂 Recommended Repository Structure

Keep your solutions organized by chapter. The automation ignores folder paths, so you can structure it however you like, but this is recommended:

```text
├── README.md
├── .github/
│   ├── scripts/
│   │   └── update_tracker.py    # The automation script
│   └── workflows/
│       └── auto-tick.yml        # The GitHub Action workflow
├── 01_Arrays_and_Hashing/
│   ├── Two_Sum.py
│   └── Contains_Duplicate.java
├── 02_Two_Pointers/
│   ├── Valid_Palindrome.cpp
│   └── Trapping_Rain_Water.py
└── ...
```

🛠️ Troubleshooting the Automation
If a problem didn't auto-tick, check the following:

```
    File Name Match: Did you name the file exactly like the problem? (e.g., Two_Sum.py for Two Sum).
    Pushed to Main: The automation only triggers on pushes to the main or master branch.
    Check the Logs: Go to the Actions tab in your GitHub repository to see the logs of the Auto-Update LeetCode Tracker workflow. It will tell you exactly which files it detected and why it might have failed to match.
```

### Final Checklist for You:

1. Copy the code above and save it as `README.md`.
2. Ensure your `.github/scripts/update_tracker.py` and `.github/workflows/auto-tick.yml` are set up (from the previous step).
3. Update the `your-username/your-repo-name` in the badge URL at the very top.
4. Push it all to GitHub and watch the magic happen on your first real solution commit!
