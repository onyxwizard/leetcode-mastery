## 🤖 How It Works

This repository uses a **GitHub Action** that scans your entire folder structure on every push.

- **File Exists?** It finds the matching problem name and checks the box `[x]`.
- **File Deleted?** It unchecks the box `[ ]` to keep the tracker 100% accurate.
- **File Extensions?** It ignores them! `1_two_sum.md`, `Two_Sum.java`, and `two-sum.py` all perfectly map to `Two Sum`.

_No special commit messages required. Just push your code!_

🛠️ Troubleshooting the Automation
If a problem didn't auto-tick, check the following:

```
    File Name Match: Did you name the file exactly like the problem? (e.g., Two_Sum.py for Two Sum).
    Pushed to Main: The automation only triggers on pushes to the main or master branch.
    Check the Logs: Go to the Actions tab in your GitHub repository to see the logs of the Auto-Update LeetCode Tracker workflow. It will tell you exactly which files it detected and why it might have failed to match.
```
